# useState：関数初期化が「いつ」実行されるかを見抜く

- 課題 ID：FE-20251227-001
- 難易度：初級（10分以内）

## 課題内容

useState に 値ではなく関数を渡したときの挙動を確認する課題。

## 要件

1. コンポーネント名：LazyInitCheck
2. 以下のように useState を定義する
   ```jsx
   const [count, setCount] = useState(() => {
     console.log('init')
     return 0
   })
   ```
3. 画面に表示するもの
   - count の値
   - 「+1」ボタン
4. 「+1」ボタンを押すと count が増える
5. console を見て、init が 何回表示されるか を確認する

## 提出物

- コンポーネントのコード
- init は何回呼ばれたか
- なぜそうなるか

## この課題の狙い（重要）

- `useState(() => initialValue)` は初期化を遅延評価するための書き
- re-render のたびに関数が実行されるわけではない
- 「関数を渡す = 毎回実行」ではない、という誤解を潰す

---

## 提出物の問いへの解答

> init は何回呼ばれたか

2回

> なぜそうなるか

stictモードであるため、2回呼ばれる。
useStateに関数を渡すと、コンポーネントが最初にマウントされるときだけその関数が実行され、戻り値が初期値として使われる。
