# useState：初期値はいつ使われる？を体感する

- 課題 ID：FE-20251226-002
- 難易度：初級（5〜10分）

## 課題内容

useState の 初期値が「最初の1回しか使われない」 ことを、
コードで体感する課題。

## 要件

1. コンポーネント名：InitialStateCheck
2. 以下の state を定義する
   - `const [count, setCount] = useState(0)`
3. 画面に以下を表示する
   - count の値
   - 「+1」ボタン
4. 「+1」ボタンを押すと count が増える
5. state の定義部分の初期値を 100 に書き換えても、途中から表示が変わらないことを確認する

※ 実装は普通のカウンターでOK

## 提出物

- コンポーネントのコード
- 余裕があれば一言だけ「なぜ初期値を変えても途中から反映されないか」

## この課題の狙い（超重要）

- useState(initialValue) は「初回レンダリング時にだけ評価される」
- re-render ≠ 再初期化
- 「render に乗る」と「初期化される」を分離して理解する

---

## 提出物の問いへの解答

> なぜ初期値を変えても途中から反映されないか

useState(initialValue) が評価されるのは初回マウント時のみ。
2回目以降の render では、React は「保存してある state」を使う。
だから、コード上で useState(0) から useState(100) に変えても既存インスタンスには影響しない。
initialValue は「初期化用」であって「現在値」ではない。
