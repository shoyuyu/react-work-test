# イベント起点の非同期を整理する

- 課題 ID：FE-20260105-001
- 課題難易度：易・ウォームアップ

## 課題内容

イベント起点の fetch を“迷わず書ける状態”にする

### 要件

- ボタンを押すと fetch を実行
- ローディング状態を表示
- 成功時：取得したデータを簡単に表示
- 失敗時：エラーメッセージを表示
- useEffect は使わない
- onClick 内で async / await

### 実装ルール（軽めだけど重要）

- state は以下の3つまで
  - isLoading
  - data
  - error
- fetch 処理は 1つの async 関数にまとめる
- try / catch / finally を使う

### 提出物

- プログラムコード
- このケースで useEffect が不要な理由（1〜2行）
- 「この実装が破綻するのはどんなケースか？」を1つ

### 評価観点

- イベント起点の非同期を 反射的に書けているか
- state の役割が混ざっていないか
- 「考えすぎていないか」

---

## 提出物の問いへの回答

> このケースで useEffect が不要な理由（1〜2行）

useEffectは状態変化に応じた自動実行が必要な時に使う。
今回のケースではボタンクリック時に一度だけfetchを実行するため。

> 「この実装が破綻するのはどんなケースか？」を1つ

アンマウント後に setState の処理が走る。
