# requestId ロジックを分離する

- 課題 ID：FE-20260128-001
- 課題難易度：（復帰用・軽め）

## 課題内容

模写で思い出す requestId ガード fetch

### 今日のゴール

- 「最後に押したボタンの結果だけ反映」できる fetch を、迷わず書ける状態に戻す
- useEffect は使わない（イベント起点）

### 模写の進め方（迷子防止・手順固定）

1. そのままコピペして表示できるところまで持っていく
2. ボタンを連打して、表示が破綻しないことを確認
3. 次の「軽い改造」を1つだけやる（下から1つ選ぶ）

### 軽い改造（どれか1つだけ）

1. randomId を useState にして、入力欄で指定できるようにする
2. Fetch failed を HTTP ${res.status} にして情報量を増やす
3. data を null に戻す「Clear」ボタンを追加

一番ラクにできそうなやつでOK（復帰最優先）
