# state を整理して書く

- 課題 ID：FE-20260107-001
- 課題難易度：易・ウォームアップ

## 課題内容

非同期 fetch に出てくる state を混ぜない

### 要件

- ボタンを押すと fetch を実行
- ローディング中は「Loading...」を表示
- 成功時：データを表示
- 失敗時：エラーメッセージを表示
- useEffect は使わない
- onClick + async/await

※ 1/5 の課題とほぼ同じ。state の置き方だけに集中。

### 実装ルール（重要だけど軽い）

- state は 必ず3つに分ける
  - isLoading: boolean
  - data: T | null
  - error: string | null
- data と error が 同時に存在しないようにする
- fetch 関数は 1 つだけ

### 提出物

- プログラムコード
- なぜ isLoading を boolean にしたか
- data と error を分けるメリットを1行で

### 評価観点

- state の責務が明確か
- if 分岐が読みやすいか
- 変に抽象化していないか

---

## 提出物の問いへの回答

> なぜ isLoading を boolean にしたか

loadingの状態はtrue/falseの2択だから

> data と error を分けるメリットを1行で

責務分離を明確に区別することで、UIロジックが簡単になる
